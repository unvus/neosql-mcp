# 자주 쓰는 슬래시 명령어

`/`로 시작하는 명령어는 입력창에 타이핑하면 자동완성 메뉴가 뜹니다.
모든 명령어 목록은 `/help`로 확인할 수 있습니다.

## 세션 관리 (가장 자주 씀)

| 명령어 | 용도 |
|--------|------|
| `/clear` | 대화·컨텍스트 초기화. **새 작업 시작 시 습관처럼 쓸 것**. 오래된 컨텍스트가 엉뚱한 결정을 유도하는 걸 막음 |
| `/compact` | 대화는 유지하면서 토큰 압축. 길어진 세션에서 종료는 싫고 컨텍스트만 줄이고 싶을 때 |
| `/resume` | 이전 세션 이어서 재개 |
| `/exit` (또는 `/quit`) | 종료 |

> **`/clear` vs `/compact`**: `/clear`는 깨끗이 비우고 새 작업, `/compact`는 같은 작업 이어가되 메모리 절약. 작업 자체를 바꾼다면 `/clear`.

## 프로젝트·환경 셋업

| 명령어 | 용도 |
|--------|------|
| `/init` | 현재 코드베이스를 읽고 `CLAUDE.md` 초안 생성 |
| `/add-dir` | 작업 디렉토리 추가. 다른 경로 파일도 읽고 쓸 수 있게 됨 |
| `/config` | 모델·테마 등 설정 UI |
| `/model` | 모델 전환 (Opus / Sonnet / Haiku) |
| `/cost` | 누적 토큰·요금 확인 |
| `/login`, `/logout` | 계정 로그인/로그아웃 |

## 메모리·컨텍스트

| 명령어 | 용도 |
|--------|------|
| `/memory` | 메모리 파일 편집기 열기 |
| `#<text>` (입력창 시작) | 텍스트를 즉시 메모리에 저장 |
| `@<filepath>` | 파일을 컨텍스트에 첨부 (탐색 비용 절약) |

## 코드 작업 보조 (내장 skill)

| 명령어 | 용도 |
|--------|------|
| `/review` | PR 코드 리뷰 |
| `/security-review` | 현재 브랜치의 보안 리뷰 |

> 내장 skill은 환경에 따라 활성화 여부가 다를 수 있고, 사용자/프로젝트 단위로 추가 skill을 등록할 수 있습니다 (`~/.claude/skills/` 또는 프로젝트의 `.claude/skills/`).

## MCP·에이전트 관리

| 명령어 | 용도 |
|--------|------|
| `/mcp` | MCP 서버 상태·설정 확인 |
| `/agents` | 서브에이전트 정의 목록·관리 |

## 도움말·기타

| 명령어 | 용도 |
|--------|------|
| `/help` | 전체 명령어 목록 |
| `/bug` | 이슈 리포트 |

## 입력창 특수 prefix (슬래시 아님)

| Prefix | 의미 |
|--------|------|
| `!<command>` | 셸 명령어 직접 실행. `gcloud auth login` 같은 인터랙티브 명령에 유용 |
| `#<text>` | 메모리에 즉시 저장 |
| `@<path>` | 파일을 컨텍스트에 첨부 |

## 워크플로 별 추천 명령어 묶음

### 새 작업 시작
```
/clear
@<관련 파일 경로 1~2개>
"<작업 지시>"
```

### 큰 기능 구현 (Plan mode 흐름)
```
/clear
Shift+Tab × 2  (Plan mode)
"X 기능 추가 계획 세워줘. 관련 파일부터 찾고."
→ 계획 검토 → 승인
→ 구현 진행
/review  (완료 후 자체 리뷰)
```

### 컨텍스트가 무거워졌을 때
```
/compact
계속 작업
```

### 작업 마무리
```
"diff 보여줘"
"커밋 메시지 초안 만들어서 커밋해줘" (명시적 지시 필요)
```

## 명령어 외워두지 마세요

`/`만 쳐도 자동완성이 뜨고, 각 명령어에 짧은 설명이 붙습니다.
**어떤 모드(Normal/Auto-accept/Plan)에서 작업 중인지** 인지하는 게 슬래시 명령어 외우기보다 훨씬 중요합니다 ([modes.md](modes.md) 참고).
