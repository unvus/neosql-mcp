# Shift+Tab 모드 (Normal / Auto-accept / Plan)

## 가장 흔한 오해

`Shift+Tab`은 **"이번 변경사항을 승인"하는 키가 아닙니다.**
세션 전체에 적용되는 **모드를 토글**하는 키이며, 한 번 누른 뒤로는 그 모드가 계속 유지됩니다.

또한 **Plan mode는 Claude Code 내장 기능**이며, 프로젝트의 `PLAN.md` 같은 사용자 정의 문서와는 아무 관련이 없습니다. 이름만 비슷할 뿐 별개 개념입니다.

## 3가지 모드

`Shift+Tab`을 누를 때마다 아래 순서로 순환합니다. 현재 어떤 모드인지는 입력창 근처 인디케이터에 표시됩니다.

```
Normal  →  Auto-accept edits  →  Plan  →  Normal  →  ...
```

| 모드 | 사용 가능 도구 | 권한 처리 | 주된 용도 |
|---|---|---|---|
| **Normal** | 모두 | 매 도구 호출마다 승인 프롬프트 | 작은 작업, 모르는 환경에서의 기본값 |
| **Auto-accept edits** | 모두 | Edit/Write 자동 승인 (셸 명령은 여전히 확인) | 신뢰 쌓인 반복 작업, 변경이 한 영역에 국한된 경우 |
| **Plan** | **읽기·검색만** (Edit/Write/실행성 Bash 차단) | 모든 조사 후 **계획서 1회 승인** | 멀티 파일 변경, 영향 범위 불확실, 처음 보는 코드베이스 |

Plan mode에서 에이전트는 조사를 끝낸 뒤 `ExitPlanMode`로 계획서를 제시합니다. 사람이 승인하면 자동으로 Normal로 빠져나가 실행을 시작하고, 거부하면 plan mode가 유지된 채 계획을 다듬는 대화로 돌아갑니다.

## Normal vs Plan — 승인 단위가 다르다

가장 헷갈리는 부분. 둘 다 "사람이 승인"하지만 **승인의 단위와 시점**이 다릅니다.

| | Normal | Plan |
|---|---|---|
| 승인 단위 | 도구 호출 **하나하나** | 전체 **계획 한 번** |
| 변경 시점 | 승인 즉시, 매 단계 | 계획 승인 후 일괄 |
| 거부 시 상태 | 앞서 승인한 변경은 이미 반영됨 | 한 글자도 변경 안 됨 |

5개 파일을 건드려야 하는 작업이라면:

- **Normal**: 파일별로 "수정할게요?" → 승인 → 즉시 수정 → 다음 파일… 중간에 거부해도 **앞서 수정된 파일은 그대로**
- **Plan**: 5개 파일 모두 분석 → 전체 계획서 제시 → 승인하면 그제서야 일괄 수정. 거부하면 **변경 0건** 상태로 계획 다듬기

비유하면 Normal은 *"단계마다 허락받고 일하는 인턴"*, Plan은 *"계획서를 결재받고 착수하는 컨설턴트"*.

## 권장 사용 패턴

### 패턴 A — 큰 작업

```
1. Shift+Tab × 2  → Plan mode 진입
2. "X 기능 추가하고 싶어. 관련 파일 찾고 구현 계획 세워줘"
3. 에이전트가 계획 제시 → 사람이 검토·수정 요청
4. 승인 → 자동으로 Normal로 빠져나옴
5. (선택) Shift+Tab × 1  → Auto-accept 켜고 빠르게 진행
6. 작업 완료 후 diff 검토 → 커밋
```

### 패턴 B — 작은 작업

```
1. Normal mode 그대로
2. "src/foo.ts:42의 typo 고쳐줘"
3. 권한 프롬프트 → 승인
```

### 패턴 C — 탐색만 필요할 때

```
1. Shift+Tab × 2  → Plan mode
2. "이 코드베이스에서 인증 처리 어디서 일어나?"
3. 답변만 받고 ESC로 plan 종료 (ExitPlanMode 없이)
```

## 모드 외 추가 단축키 메모

- **ESC** — 진행 중인 에이전트 응답 즉시 중단
- **ESC ESC** (빠르게 두 번) — 직전 사람 메시지로 돌아가서 수정 (대화 분기)
- **Ctrl+C** — 입력 중인 텍스트 취소 / 한 번 더 누르면 종료
- **!`<command>`** — 입력창 맨 앞에서 `!`로 시작하면 셸 직접 실행 (인터랙티브 명령용)
- **#`<text>`** — `#`으로 시작하면 메모리에 즉시 저장

## 참고

- 모드 인디케이터 위치: 입력창 우측 또는 하단 (버전에 따라 다름)
- Plan mode에서 `ExitPlanMode`가 호출되면 사람의 승인 다이얼로그가 뜨고, 거기서 거부하면 plan mode가 유지됩니다
- 자동 승인 모드(`auto-accept`)는 위험할 수 있으니, 신뢰 안 가는 외부 입력을 처리할 때(웹 페치한 내용 기반 편집 등)는 끄는 편이 안전
